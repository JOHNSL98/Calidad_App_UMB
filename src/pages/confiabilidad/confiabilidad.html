Confiabilidad

La confiabilidad es la capacidad del software para asegurar un nivel de funcionamiento adecuado cuando es utilizando en condiciones específicas. En este caso al confiabilidad se amplia sostener un nivel especificado de funcionamiento y no una función requerida.

Madurez

La capacidad que tiene el software para evitar fallas cuando encuentra errores. Ejemplo, la forma como el software advierte al usuario cuando realiza operaciones en la unidad de diskett vacia, o cuando no encuentra espacio suficiente el disco duro donde esta almacenando los datos.

¿Fueron detectados defectos durante el periodo de ejecucion del sistema? ¿El sistema notifica o advierte cuando el usuario realiza cambios cobre una unidad de almacenamiento vacia? ¿El sistema notifica o advierte cuando el sistema no encuentra disponibildad de almacenamiento suficiente en la unidad previamente seleccionada por el usuario? ¿El sistema cuenta con la capacidad de corregir defeectos? Tolerancia a errores

La capacidad que tiene el software para mantener un nivel de funcionamiento en caso de errores.

¿Puede mantener el sistema el nivel de rendimiento despues de un fallo por cierto tiempo? ¿El sistema sigue funcionando después de un fallo pero con aspectos de funcionalidad deteriorados? ¿Despues de un error se establece una parada segura que no produce efectos en la tolerancia respecto a otros componentes del sistema? ¿El sistema cuenta con mecanismos de confirmación para evitar que acciones irreversibles sean llevadas acabo?

Recuperabilidad

La capacidad que tiene el software para restablecer su funcionamiento adecuado y recuperar los datos afectados en el caso de una falla.
